# Bank 66: Empty / Garbage Data
## Overview
**Local Hex Range:** `0x4000` - `0x7FFF`  
**Global Hex Range:** `0x198000` - `0x19BFFF`  
**Bank Type:** N/A  
**Category:** Empty / Garbage  
**Summary:** Leftover data from Wario Land II. Entire bank matches Bank `0x66` from WL2.

---

## Memory Map (Granular)
*Break down the specific blocks within this bank.*

| Offset (Local) | Offset (Global) | Length | Description |
| :------------- | :-------------- | :----- | :---------- |
| `0x4000` | `0x198000` | `0xC80` | Wario Land II sprite data. Standard `2bpp` format, `FC/NES x 16` / `1.2H` Pattern. |
| `0x4C80` | `0x198C80` | `0xB80` | 0x00's |
| `0x5800` | `0x199800` | `0x8E8` | Wario Land II sprite data. Stored in a `2bpp` format, possibly with some compression<sup>[1](#note1)</sup>, `FC/NES x 16` / `1.2H` Pattern. |
| `0x60E8` | `0x19A0E8` | `0x1F18` | 0x00's |

<span style="font-size: 11px;"><a id="note1"></a>**[1]** Did not test DWM decompression. However, since entire bank matches, it is highly unlikely it is the same algorithm if compressed. Visually this sector does not follow the same tendencies. </span>
